## PostgreSQLのバックアップについて Addonのpgbackupsを使用することで、毎日自動的にバックアップを取得してくれます。 ``` heroku addons:add pgbackups:auto-month ``` pgbackupsにはPlus, One Week Retention, One Month Retentionの3つのプランがありますがどれも無料なので特に理由がない限りバックアップ保持数の多いOne Month Retentionを使用すると良いです。 ### バックアップの一覧を表示する ``` $ heroku pgbackups ID Backup Time Status Size Database ---- ------------------------- 上記は2014年7月1日の実行結果ですが、Dailyのバックアップが1週間分、Weeklyのバックアップが一ヶ月分、手動バックアップが２世代分保持されていることが分かります。 ### 手動でバックアップを取る ``` heroku pgbackups:capture [DATABASE] ``` DATABASEには環境変数のHEROKU_POSTGRESQL_XXXX_URLを指定します。(XXXXの部分にはランダムに色名が設定されます。) DATABASEを省略した場合はDATABASE_URLが対象になります。 ### バックアップからリストアする ``` heroku pgbackups:restore [ [BACKUP_ID|BACKUP_URL]] ``` DATABASEには環境変数のHEROKU_POSTGRESQL_XXXX_URLを指定します。(XXXXの部分にはランダムに色名が設定されます。) DATABASEを省略した場合はDATABASE_URLが対象になります。 BACKUP_ID(またはBACKUP_URL)を省略した場合は最新のバックアップが対象になります。 ### バックアップからダウンロードする ``` heroku pgbackups:url [BACKUP_ID] ``` 指定のバックアップをダウンロードするためのURLを表示します。 URLの有効期間は10分間なのでその間にブラウザやcurl等のツールを用いてダウンロードしてください。 BACKUP_ID(またはBACKUP_URL)を省略した場合は最新のバックアップが対象になります。 ダウンロードしたバックアップファイルはpg_dumpコマンドでローカルのPostgreSQLにインポートすることができます。
