## サンプルの説明 適当なメールをtest@ht.flectdev.com宛にを送信してください。 受信した最新のメールを画面に表示します。(メール送信後は画面をリロードしてください。) ----------------------------------------- ## 解説 SendGridのParse APIを使用してメール受信時に任意の処理を実行します。

このサンプルを実行するには[SendGrid](http://sendgrid.com/)アドオンが必要です

Addonの追加コマンドは以下です。(Starterは無料)

``` heroku addons:add sendgrid ```

Addonを追加すると環境変数に**SENDGRID_USERNAME**, **SENDGRID_PASSWORD**という二つの変数が設定される
### 仕組み メール着信時にはその内容がMultiData/Form形式で指定のURLにPOSTされるのでその内容をパースすることで添付ファイルを含むメールの内容を取得することができます。 （このサンプルではメールのパースに[Commons FileUpload](http://commons.apache.org/proper/commons-fileupload/index.html)を使用しています。） ### Parse APIの設定 Parse APIを使用するためにはまずDNS側でSendGridで受けるメールのドメイン(サブドメイン)のMXレコードを「mx.sendgrid.net」に設定します。 この設定により指定のドメイン宛に送られてきたメールはすべてSendGrid宛に送られるようになります。 次にSendGridの管理画面から Developers ＞ Parsing Incoming Emails の画面を表示して - Hostname: DNSでSendGridでメールを受けるように設定したドメイン - Url: メール着信時にHTTP POSTを実行するURL を指定します。 このサンプルでは - Hostname: ht.flectdev.com - Url: http://flect-heroku-sample.herokuapp.com/inboundMail を指定しています。 サンプルでは「test@ht.flectdev.com」にメールを送るように指示していますが、実際にはローカルパート部分には何を指定してもメールは届きます。 実際にメール着信時にどのような設定を行っているかは関連ソースをご確認ください。 ### Parse API使用時の注意事項 DNSの設定によりSendGridで受けたメールはURL通知が行われるだけでどこにも保存されません。 またDNS側の設定を行っているにも関わらずSendGrid側で通知URLを設定していない状態では任意の第3者がそのドメインに対する通知URLを設定することができてしまいます。 ご利用の際には必ずDNS側の設定とSendGrid側の設定をセットで行うようにしてください。 ### 関連ソース(ソースの閲覧には権限が必要です。) - [InboundMailServlet.java](https://github.com/FLECT-DEV-TEAM/heroku-samples/blob/master/src/main/java/herokuSamples/web/mail/InboundMailServlet.java)
